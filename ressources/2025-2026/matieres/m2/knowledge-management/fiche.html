1. INTRODUCTION AU KNOWLEDGE MANAGEMENT

Définition

Le Knowledge Management (KM) est l'ensemble des méthodes, outils, processus et technologies permettant de :

  • Créer des connaissances
  • Capturer des connaissances
  • Structurer des connaissances
  • Stocker des connaissances
  • Partager des connaissances
  • Réutiliser des connaissances
  • Valoriser des connaissances

dans le but d'améliorer :

  • la performance de l'organisation ;
  • l'innovation ;
  • la prise de décision ;
  • l'apprentissage organisationnel ;
  • la compétitivité.

Pourquoi faire du KM ?

Sans KM :

  • perte de connaissances lors des départs ;
  • répétition des erreurs ;
  • duplication du travail ;
  • faible collaboration ;
  • difficulté à prendre des décisions.

Avec KM :

  • conservation du savoir ;
  • amélioration de la productivité ;
  • accélération de la prise de décision ;
  • meilleure innovation ;
  • réduction des coûts.

3. TYPES DE CONNAISSANCES

Connaissance explicite

Connaissance formalisée.

Exemples :

  • procédures ;
  • manuels ;
  • documents ;
  • bases de données ;
  • rapports.

Facile à stocker et partager.

Connaissance tacite

Connaissance difficile à formaliser.

Exemples :

  • expérience ;
  • intuition ;
  • savoir-faire ;
  • expertise métier.

Exemple :

Un urbaniste expérimenté qui sait détecter
rapidement un dossier problématique.

Pourquoi la connaissance tacite est importante ?

Selon plusieurs études :

80 à 90 % des connaissances
d'une organisation sont tacites.

Le principal objectif du KM est souvent :

Transformer le tacite en explicite.

4. MODÈLE SECI DE NONAKA

Transformation de la connaissance.

1. Socialisation (Tacite → Tacite)

Transmission directe.

Exemples :

  • mentorat ;
  • compagnonnage ;
  • observation.

2. Externalisation (Tacite → Explicite)

Formalisation du savoir.

Exemples :

  • rédaction de guides ;
  • interviews d'experts ;
  • cartographie cognitive.

3. Combinaison (Explicite → Explicite)

Fusion de connaissances existantes.

Exemple :

Fusionner plusieurs rapports
pour créer une procédure.

4. Internalisation (Explicite → Tacite)

Apprentissage.

Exemple :

Lecture d'un guide puis application pratique.

10. RDF (RESOURCE DESCRIPTION FRAMEWORK)

Définition

Modèle de représentation des connaissances.

Principe

RDF représente tout sous forme de triplets.

Sujet - Prédicat - Objet

Exemple 

Arnaud  aime  LaJoconde

Sujet :

Arnaud

Prédicat :

aime

Objet :

LaJoconde

Graphe RDF

Chaque triplet devient :

Noeud → Relation → Noeud

Exemple :

Arnaud ----aime----> LaJoconde

11. RDF/XML

Définition

Syntaxe XML permettant d'écrire du RDF.

Exemple 

<rdf:Description rdf:about="Arnaud">
    <ex:aime rdf:resource="LaJoconde"/>
</rdf:Description>

12. RDF SCHEMA (RDFS)

Permet d'ajouter :

  • classes ;
  • hiérarchies ;
  • propriétés.

Classe

Exemple :

Personne

Sous-classe 

Etudiant
      ⬇
Personne

En RDF :

Etudiant rdfs:subClassOf Personne

Inférence

Si :

Paul rdf:type Etudiant

et

Etudiant rdfs:subClassOf Personne

alors :

Paul rdf:type Personne

14. OWL (WEB ONTOLOGY LANGUAGE)

Définition

Langage permettant de construire des ontologies riches.

Plus puissant que RDF et RDFS.

owl:Class

Déclaration d'une classe.

Médecin
Patient

owl:equivalentClass

Deux classes sont identiques.

Cardiologue ≡ MédecinCardiaque

owl:disjointWith

Deux classes incompatibles.

Patient disjointWith Médicament

Un individu ne peut pas appartenir aux deux.

15. RESTRICTIONS OWL

Restriction d'existence 

Patient
   doit avoir
au moins un médecin

Notation :

Patient
⊑ consulte some Médecin

Restriction de cardinalité

Exemple :

Personne possède exactement 1 date de naissance.

Exemple

    sous-classe de

17. SPARQL

Définition

Langage de requête du Web sémantique.

Equivalent du SQL pour RDF.

Structure 

SELECT ?x
WHERE {
    ...
}

Exemple 1

Lister tous les étudiants.

SELECT ?x
WHERE {
   ?x rdf:type ex:Etudiant
}

Exemple 2

Trouver les salles réservées.

SELECT ?salle
WHERE {
   ?reservation ex:concerneSalle ?salle
}

FILTER

Filtrer les résultats.

FILTER(?age > 18)

LIMIT

Limiter les résultats.

LIMIT 10

INSERT DATA

Ajouter des triplets.

INSERT DATA {
   ex:Paul rdf:type ex:Etudiant
}

Q1. Quel est l'objectif principal du Web Sémantique ?

Réponse

✅ Permettre aux machines de comprendre et traiter les données du Web.

Explication

Le Web traditionnel permet aux humains de lire les informations.

Le Web Sémantique ajoute :

  • du sens ;
  • des relations ;
  • une structure formelle.

afin que les machines puissent également comprendre les données.

Q2. Quelle est la couche la plus basse de l'architecture du Web Sémantique ?

Réponse

✅ URI / Unicode

Explication

Toutes les autres couches reposent sur l'identification unique des ressources.

Exemple :

http://example.org/person/arnaud

Sans URI, impossible d'identifier précisément les ressources.

Q3. Quel langage est utilisé pour représenter les connaissances du Web Sémantique ?

Réponse

✅ RDF

Explication

RDF est le modèle fondamental de représentation.

Toutes les informations sont exprimées sous forme de :

Sujet - Prédicat - Objet

Explication

  • de faire des inférences.

C'est le langage d'ontologie du Web Sémantique.
