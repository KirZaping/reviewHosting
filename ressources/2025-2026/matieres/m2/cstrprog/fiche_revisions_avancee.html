Programmation Par Contraintes — Révisions Avancées

1. Objectif des optimisations CSP

Un solveur CSP explore un espace de recherche potentiellement gigantesque.

Pour un problème avec :

  • nnn variables,
  • domaine moyen ∣D∣|D|∣D∣,

le nombre de possibilités peut atteindre :

∣D∣n|D|^n∣D∣n

L’objectif des optimisations est donc de :

Réduire l’espace de recherche le plus tôt possible

2. Propagation de contraintes

2.1 Définition

La propagation consiste à :

Retirer des valeurs impossibles des domaines

avant ou pendant la recherche.

2.2 Exemple

Variables :

X∈{1,2,3}X \in \{1,2,3\}X∈{1,2,3}

Y∈{1}Y \in \{1\}Y∈{1}

Contrainte :

X<YX < YX<Y

Alors :

X impossible

Le solveur détecte immédiatement l’incohérence.

2.3 Objectif

Réduire :

  • le nombre de branches,
  • le backtracking,
  • le temps de résolution.

3. Consistency (Cohérence)

3.1 Définition générale

Une cohérence cherche à garantir qu’un domaine :

ne contient pas de valeurs impossibles

selon certaines règles locales.

4. Arc Consistency (AC)

4.1 Définition intuitive

Une variable est arc-cohérente si :

Chaque valeur de son domaine possède
au moins un support compatible
dans les autres variables

4.2 Exemple

X∈{1,2,3}X \in \{1,2,3\}X∈{1,2,3}

Y∈{2}Y \in \{2\}Y∈{2}

Contrainte :

X<YX < YX<Y

Analyse

  • X=1X=1X=1 possible,
  • X=2X=2X=2 impossible,
  • X=3X=3X=3 impossible.

Résultat

X∈{1}X \in \{1\}X∈{1}

4.3 Caractéristiques

Avantages Inconvénients
Très forte propagation Coût élevé
Réduction forte des domaines Peu scalable
Moins de backtracking Plus lent localement

4.4 Utilisation typique

Très efficace pour :

  • Sudoku,
  • graph coloring,
  • CSP discrets,
  • petits domaines.

5. Bound Consistency (BC)

5.1 Définition

La Bound Consistency vérifie uniquement :

  • borne minimale,
  • borne maximale.

Elle ne vérifie pas toutes les valeurs internes.

5.2 Principe

Une variable est BC si :

ses bornes possèdent un support compatible

5.3 Exemple

X∈[1,10]X \in [1,10]X∈[1,10]

Y∈[1,5]Y \in [1,5]Y∈[1,5]

Contrainte :

X<YX < YX<Y

Propagation

Alors :

X∈[1,4]X \in [1,4]X∈[1,4]

5.4 Caractéristiques

Avantages Inconvénients
Très rapide Propagation plus faible
Très scalable Plus de recherche
Adaptée grands domaines Moins précise

5.5 Utilisation typique

Très utilisée pour :

  • scheduling,
  • optimisation industrielle,
  • contraintes numériques,
  • grands domaines.

7. Backtracking

7.1 Principe

Le solveur :

  1. choisit une variable,
  2. teste une valeur,
  3. propage,
  4. revient en arrière si contradiction.

7.2 Objectif

Explorer intelligemment :

l’espace de recherche

8. Heuristiques de recherche

8.1 Pourquoi ?

Le choix :

  • des variables,
  • des valeurs,

impacte énormément les performances.

8.2 First Fail

Très célèbre.

Principe :

Choisir la variable au plus petit domaine

Pourquoi ?

Car elle a le plus de chances :

d’échouer rapidement

8.3 Choco 

solver.setSearch(
    Search.minDomLBSearch(vars)
);

9. Symmetry Breaking

9.1 Problème

Certaines solutions sont équivalentes :

  • miroir,
  • rotation,
  • permutation.

Le solveur explore inutilement :

les mêmes structures

9.2 Solution

Ajouter des contraintes supprimant les symétries.

Exemple 

model.arithm(q[0], "<", q[n-1]).post();

9.3 Effet

Réduction importante :

  • du temps,
  • du nombre de branches,
  • du backtracking.

10. Contraintes globales

10.1 Définition

Une contrainte globale possède :

un algorithme spécialisé de propagation

10.2 Exemple majeur

AllDifferent 

model.allDifferent(vars).post();

10.3 Pourquoi elles sont importantes

Préférer :

allDifferent(vars)

plutôt que :

x1 != x2
x1 != x3
...

car :

  • propagation beaucoup plus forte,
  • algorithmes optimisés.

11. Optimisation — COP

11.1 CSP vs COP

CSP

Trouver :

une solution valide

COP

Trouver :

la meilleure solution

11.2 Objectifs classiques

  • minimiser un coût,
  • maximiser un profit,
  • minimiser une durée,
  • minimiser un nombre de créneaux.

11.3 Définition d’objectif

Minimisation 

model.setObjective(
    Model.MINIMIZE,
    objective
);

Maximisation 

model.setObjective(
    Model.MAXIMIZE,
    objective
);

14. Randomisation

14.1 Pourquoi ?

Les heuristiques déterministes peuvent :

explorer toujours les mêmes mauvaises branches

14.2 Objectif

Diversifier :

  • les choix,
  • les branches,
  • les explorations.

14.3 Techniques

  • choix aléatoire de variable,
  • choix aléatoire de valeur,
  • random restart.

15. Random Restart

15.1 Principe

Arrêter puis recommencer :

avec une stratégie différente

15.2 Intérêt

Très efficace pour :

  • gros CSP,
  • optimisation difficile,
  • problèmes avec minima locaux.

16. Trade-off fondamental du CSP

Toute optimisation CSP équilibre :

Temps de propagation
VS
Temps de recherche

17. Réduction des domaines

17.1 Principe

Plus les domaines sont petits :

  • plus la propagation est forte,
  • plus le solveur est rapide.

17.2 Bonne pratique

Toujours chercher :

les domaines les plus précis possibles
