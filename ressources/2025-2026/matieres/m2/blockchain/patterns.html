Blockchain Patterns - Fiche de Révision Complète

Introduction

Les Blockchain Patterns sont des solutions d'architecture réutilisables permettant de résoudre des problèmes récurrents rencontrés lors de la conception d'applications blockchain.

L'objectif est de savoir :

  • Quel pattern utiliser
  • Pourquoi l'utiliser
  • Quels avantages il apporte
  • Quels inconvénients il introduit

1. Interaction With External World

Oracle Pattern

Définition

Permet à une blockchain d'accéder à des informations provenant du monde extérieur.

Un smart contract ne peut pas accéder directement à Internet.

Problème

Comment récupérer une donnée externe ?

  • météo
  • prix d'une action
  • résultat sportif
  • position GPS

Solution

Un oracle récupère la donnée et la transmet à la blockchain.

Exemple

Assurance agricole :

Si la pluviométrie est inférieure à 10 mm, alors indemniser automatiquement l'agriculteur.

L'oracle météo transmet la mesure au smart contract.

Avantages

  • Connecte la blockchain au monde réel

Inconvénients

  • Introduit un tiers de confiance

À retenir

Oracle = Blockchain ← Monde extérieur

Reverse Oracle

Définition

Permet à un système externe d'utiliser les données présentes sur la blockchain.

Problème

Comment un logiciel existant peut-il consulter la blockchain ?

Solution

Le système externe interroge le smart contract.

Exemple

Une entreprise consulte les absences d'un apprenti enregistrées sur blockchain.

Avantages

  • Intégration avec les systèmes existants

Inconvénients

  • Nécessite une adaptation des applications existantes

À retenir

Reverse Oracle = Monde extérieur ← Blockchain

Legal & Smart Contract Pair

Définition

Association d'un contrat juridique classique avec un smart contract.

Problème

Les smart contracts ne couvrent pas toujours tous les aspects juridiques.

Solution

Conserver :

  • un contrat légal
  • un smart contract

Les deux se référencent mutuellement.

Exemple

Bail immobilier :

  • contrat PDF signé
  • smart contract qui gère automatiquement les loyers

Avantages

  • Automatisation
  • Traçabilité

Inconvénients

  • Traduire un texte juridique en code est complexe

À retenir

Contrat juridique + Smart Contract = même engagement

2. Data Management Patterns

Encrypting On-Chain Data

Définition

Chiffrer les données avant leur stockage sur blockchain.

Problème

Tout le monde peut lire les données stockées sur une blockchain publique.

Solution 

Donnée
   ↓
Chiffrement
   ↓
Blockchain

Exemple

Dossier médical.

Avantages

  • Confidentialité

Inconvénients

  • Gestion des clés
  • Révocation difficile

À retenir

Données sensibles → Chiffrement obligatoire

Tokenization

Définition

Représenter un actif réel ou numérique sous forme de token.

Problème

Comment représenter un actif sur blockchain ?

Solution

Créer un token représentant cet actif.

Exemples

  • Immobilier
  • Or
  • Actions
  • Billets d'événements
  • NFT

Exemple

Appartement de 500 000 € :

1000 tokens
=
0,1 % du bien chacun

Avantages

  • Fractionnement
  • Liquidité

Inconvénients

  • La propriété légale n'est pas automatiquement garantie

À retenir

Tokenisation = représentation numérique d'un actif

Off-Chain Data Storage

Définition

Stocker les gros fichiers hors blockchain et uniquement leur hash sur blockchain.

Problème

Le stockage sur blockchain est coûteux et limité.

Solution 

Fichier PDF
     ↓
SHA-256
     ↓
Hash
     ↓
Blockchain

Exemple

Certificat médical.

Vérification 

Hash actuel ≠ Hash blockchain
→ Document modifié

Avantages

  • Faible coût
  • Très scalable

Inconvénients

  • Le fichier peut être perdu hors blockchain

À retenir

Blockchain stocke la preuve, pas forcément le document

State Channel

Définition

Effectuer plusieurs transactions hors blockchain puis enregistrer uniquement le résultat final.

Problème

Les transactions blockchain sont lentes et coûteuses.

Solution 

1000 transactions privées
          ↓
1 transaction finale blockchain

Exemples

  • Lightning Network
  • Raiden Network

Avantages

  • Très rapide
  • Très peu coûteux

Inconvénients

  • Complexité plus élevée

À retenir

State Channel = blockchain au début et à la fin uniquement

3. Security Patterns

Multiple Authorization

Définition

Plusieurs signatures sont nécessaires pour valider une opération.

Principe

M signatures parmi N sont requises.

Notation :

M-of-N

Exemple 

2 signatures nécessaires parmi :

- Directeur
- Comptable
- DSI

Exemple :

2-of-3 Multisig

Avantages

  • Réduit le risque de fraude
  • Tolère la perte d'une clé

Inconvénients

  • Les autorités doivent être connues à l'avance

À retenir

Multisig = plusieurs validations nécessaires

Off-Chain Secret Enabled Dynamic Authorization

Définition

Autorisation basée sur un secret partagé hors blockchain.

Aussi appelé :

Hashlock

Problème

On ne connaît pas à l'avance la personne qui exécutera l'action.

Solution

Le hash du secret est stocké sur blockchain.

Celui qui révèle le secret obtient le droit d'exécuter l'opération.

Exemple

Lightning Network.

Avantages

  • Très flexible

Inconvénients

  • Secret utilisable une seule fois

À retenir

Le secret devient l'autorisation

X-Confirmation

Définition

Attendre plusieurs blocs avant de considérer une transaction comme définitive.

Pourquoi ?

Une transaction récente peut disparaître lors d'un fork.

Exemple

Bitcoin :

6 confirmations

Avantages

  • Réduit fortement le risque de réorganisation

Inconvénients

  • Augmente le temps d'attente

À retenir

Plus il y a de confirmations, plus la transaction est sûre

4. Structural Smart Contract Patterns

Contract Registry

Définition

Annuaire des smart contracts.

Problème

Les smart contracts sont immuables.

Solution 

Nom logique
      ↓
Adresse du contrat

Exemple 

AttendanceContract
      ↓
0x123...

Avantages

  • Permet les mises à jour

Inconvénients

  • Ajoute un niveau d'indirection

À retenir

Registry = annuaire des contrats

Embedded Permission

Définition

Gestion des permissions directement dans le smart contract.

Exemple 

onlyOwner
onlyAdmin
onlyProfessor

Avantages

  • Contrôle d'accès précis

Inconvénients

  • Complexifie le contrat

À retenir

Embedded Permission = contrôle d'accès intégré

Data Contract

Définition

Séparer :

  • logique métier
  • données

Structure 

Business Contract
        ↓
Data Contract

Avantages

  • Facilite les mises à jour

Inconvénients

  • Architecture plus complexe

À retenir

Séparation logique / données

Factory Contract

Définition

Smart contract capable de créer d'autres smart contracts.

Exemple 

Factory
   ↓
NFT A
NFT B
NFT C

Avantages

  • Déploiement automatisé

Inconvénients

  • Coût supplémentaire

À retenir

Factory = générateur de contrats

Incentive Execution

Définition

Récompenser financièrement les utilisateurs qui exécutent certaines fonctions.

Exemple 

Récompense :
0.01 ETH

pour l'utilisateur qui déclenche une fonction périodique.

Avantages

  • Encourage la participation

Inconvénients

  • Coût supplémentaire

À retenir

Récompenser l'exécution pour assurer la maintenance
