Programmation Par Contraintes (CSP) avec Choco Solver

1. Introduction au CSP

1.1 Définition d’un CSP

Un CSP (Constraint Satisfaction Problem) est défini par :

CSP=(X,D,C)CSP = (X, D, C)CSP=(X,D,C)

avec :

  • XXX : ensemble des variables
  • DDD : domaines des variables
  • CCC : ensemble des contraintes

1.2 Objectif d’un CSP

Le but est de trouver une affectation de variables qui satisfait toutes les contraintes.

1.3 Philosophie de la programmation par contraintes

On ne programme pas :

COMMENT trouver la solution

On décrit :

QUELLES propriétés la solution doit respecter

Le solveur se charge automatiquement :

  • de la propagation,
  • de la recherche,
  • du backtracking,
  • de l’exploration des solutions.

2. Construction d’un CSP

2.1 Définir les variables

Les variables représentent les décisions du problème.

Exemples

  • heure d’une tâche,
  • salle attribuée,
  • valeur d’une case Sudoku.

2.2 Définir les domaines

Le domaine correspond aux valeurs possibles d’une variable.

Exemple par ensemble 

X ∈ {1,2,3}

Exemple par intervalle 

X ∈ [1,10]

2.3 Ajouter les contraintes

Les contraintes expriment les règles du problème.

Exemples 

X < Y
A + B = 10
AllDifferent(A,B,C)

2.4 Résoudre

Le solveur recherche automatiquement une solution valide.

3. Fonctionnement du solveur

3.1 Propagation de contraintes

Le solveur réduit les domaines des variables grâce aux contraintes.

Exemple 

X ∈ {1,2,3}
Y ∈ {1,2,3}
X < Y

Si :

X = 3

alors :

Y impossible

Le solveur détecte immédiatement l’incohérence.

3.2 Recherche et Backtracking

Le solveur :

  1. choisit une variable,
  2. tente une valeur,
  3. propage les contraintes,
  4. revient en arrière si contradiction.

4. Types de contraintes

4.1 Contraintes arithmétiques

Exemples 

X + Y = 10
X < Y

4.2 Contraintes logiques

Exemple 

A => B

4.3 Contraintes AllDifferent

Toutes les variables doivent être différentes.

Utilisation classique

  • Sudoku,
  • emplois du temps,
  • affectation de ressources.

9. Structure minimale complète Choco

9.1 Exemple complet 

import org.chocosolver.solver.Model;
import org.chocosolver.solver.Solver;
import org.chocosolver.solver.variables.IntVar;

public class Main {

    public static void main(String[] args) {

        // 1. Création du modèle
        Model model = new Model("Mon CSP");

        // 2. Variables
        IntVar x = model.intVar("X", 1, 3);
        IntVar y = model.intVar("Y", 1, 3);

        // 3. Contraintes
        model.arithm(x, "<", y).post();

        // 4. Solveur
        Solver solver = model.getSolver();

        // 5. Résolution
        while (solver.solve()) {

            System.out.println(
                "X = " + x.getValue()
                + ", Y = " + y.getValue()
            );
        }
    }
}

10. Exemple complet de modélisation

10.1 Énoncé

3 tâches doivent être planifiées :

  • heures entre 1 et 5,
  • pas au même moment,
  • A avant B.

10.2 Modélisation CSP

Variables 

A, B, C

Domaines 

A,B,C ∈ [1,5]

Contraintes 

AllDifferent(A,B,C)
A < B

10.3 Traduction Choco 

Model model = new Model("Planning");

IntVar A = model.intVar("A", 1, 5);
IntVar B = model.intVar("B", 1, 5);
IntVar C = model.intVar("C", 1, 5);

model.allDifferent(new IntVar[]{A,B,C}).post();

model.arithm(A, "<", B).post();

Solver solver = model.getSolver();

while(solver.solve()) {

    System.out.println(
        "A=" + A.getValue()
        + " B=" + B.getValue()
        + " C=" + C.getValue()
    );
}

11. COP — Constraint Optimization Problem

11.1 Définition

Un COP cherche la meilleure solution possible.

Contrairement à un CSP classique qui cherche seulement :

  • une solution valide,

un COP cherche :

  • la solution optimale.

11.2 Exemples d’objectifs

  • minimiser un coût,
  • maximiser un profit,
  • minimiser une durée,
  • maximiser une satisfaction.

11.3 Définition de l’objectif

Minimisation 

model.setObjective(Model.MINIMIZE, cost);

Maximisation 

model.setObjective(Model.MAXIMIZE, profit);
