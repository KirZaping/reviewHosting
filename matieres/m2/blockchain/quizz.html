Paiements et Transactions

Comment fonctionne un paiement Bitcoin ?

Pour transférer un bitcoin, le propriétaire doit :

  • Signer avec sa clé privée
  • Le hash de la transaction précédente
  • La clé publique du futur propriétaire

Réponse QCM

✅ Le propriétaire signe avec sa clé privée le hash de la transaction précédente et la clé publique du prochain détenteur.

Peut-on envoyer une transaction non signée ?

❌ Non.

Toute transaction doit être signée afin de prouver la propriété des fonds.

Réponse QCM

❌ Faux

Double Spending

Définition

Le double spending consiste à dépenser la même pièce plusieurs fois.

Réponse QCM

✅ La même pièce est utilisée pour payer deux achats simultanément.

Comment Nakamoto résout le double spending ?

Le réseau entier s'accorde sur un ordre unique des transactions.

Réponse QCM

✅ Tous les acteurs du système s'accordent sur l'ordre des transactions.

Comment vérifier l'authenticité d'une pièce ?

Le destinataire doit :

  • Vérifier chaque transaction précédente
  • Vérifier chaque signature
  • Vérifier toute la chaîne de possession

Réponse QCM

✅ Vérifier toute la chaîne de possession de la pièce.

Validation des Blocs

Pourquoi un bloc peut être invalide ?

Un bloc est invalide si :

  • Son hash est incorrect
  • Le Proof of Work est insuffisant
  • Une transaction est invalide

Réponses QCM

✅ Nombre insuffisant de zéros dans le hash

✅ Transaction invalide
