2. ISO 27001

Définition

Norme internationale de management de la sécurité de l'information.

Objectif :

Mettre en place un SMSI
(Security Management System / ISMS)

SMSI (ISMS)

Système permettant :

  • identifier les risques
  • analyser les risques
  • traiter les risques
  • surveiller les risques

3. LES CLAUSES ISO 27001

Les clauses 1 à 3 sont informatives.

Les clauses 4 à 10 sont certifiantes.

Clause 1 - Scope

Décrit le périmètre de la norme.

Clause 2 - Normative References

Références normatives.

Clause 3 - Terms and Definitions

Définitions officielles.

Clause 4 - Context of the Organization

Question fréquente d'examen.

Objectif :

Comprendre l'environnement.

Analyse :

  • parties prenantes
  • contraintes internes
  • contraintes externes
  • réglementation

Document associé :

  • Scope Statement

Responsable :

  • Direction
  • RSSI

Clause 5 - Leadership

Objectif :

Montrer l'engagement de la direction.

La direction doit :

  • définir la politique SSI
  • fournir les ressources
  • soutenir le SMSI

Documents :

  • Information Security Policy

Responsable :

  • Top Management

Clause 6 - Planning

Objectif :

Préparer la gestion des risques.

Documents :

  • Risk Assessment Methodology
  • Risk Register
  • Risk Treatment Plan

Responsable :

  • RSSI
  • Risk Manager

Clause 7 - Support

Objectif :

Donner les moyens nécessaires.

Contient :

  • compétences
  • formation
  • sensibilisation
  • communication
  • documentation

Documents :

  • Training Plan
  • Awareness Program

Clause 8 - Operation

Objectif :

Faire fonctionner le SMSI.

Documents :

  • procédures
  • plans opérationnels

Clause 9 - Performance Evaluation

Question de QCM fréquente.

Objectif :

Mesurer l'efficacité.

Contient :

  • audits internes
  • KPI
  • revues de direction

Documents :

  • Audit Report
  • Management Review

Clause 10 - Improvement

Objectif :

Amélioration continue.

Contient :

  • actions correctives
  • traitement des incidents

Documents :

  • Corrective Action Plan

5. GESTION DES RISQUES

Définition

Un risque est :

Risque = Menace × Vulnérabilité × Impact

Méthode complète

1. Identifier l'actif

Exemples :

  • serveur
  • VPN
  • base de données
  • SharePoint

2. Identifier la menace

Exemples :

  • ransomware
  • phishing
  • DDoS

3. Identifier la vulnérabilité

Exemples :

  • absence MFA
  • mot de passe faible
  • serveur non patché

4. Évaluer

Probabilité

et

Impact

5. Définir le niveau de risque

Matrice de risque

Impact / Probabilité Faible Moyen Fort
Faible Bas Bas Moyen
Moyen Bas Moyen Élevé
Fort Moyen Élevé Critique

Traitement des risques

Réduire

Ex :

MFA

Éviter

Supprimer l'activité

Transférer

Assurance

Accepter

Risque faible

6. OWASP TOP 10 (2021)

A01 - Broken Access Control

Accès à des ressources non autorisées.

Ex :

modifier un ID dans une URL.

A02 - Cryptographic Failures

Protection cryptographique insuffisante.

Ex :

MD5
mot de passe non chiffré

A03 - Injection

Données utilisateur exécutées comme commande.

Ex :

SQL Injection

A04 - Insecure Design

Erreur de conception.

Ex :

aucun contrôle métier prévu.

A05 - Security Misconfiguration

Mauvaise configuration.

Ex :

admin/admin

A06 - Vulnerable and Outdated Components

Logiciels vulnérables.

Ex :

Apache non patché.

A07 - Identification and Authentication Failures

Authentification faible.

Ex :

absence MFA

A08 - Software and Data Integrity Failures

Chaîne logicielle compromise.

Ex :

pipeline CI/CD non sécurisé.

A09 - Security Logging and Monitoring Failures

Logs insuffisants.

Ex :

aucune détection d'intrusion.

A10 - SSRF

Le serveur effectue une requête vers une cible choisie par l'attaquant.

Ex :

accès au metadata AWS.

7. RGPD

Définition

Règlement européen de protection des données personnelles.

Donnée personnelle

Toute information permettant d'identifier une personne.

Ex :

  • nom
  • email
  • téléphone
  • IP

Données sensibles

Protection renforcée.

Ex :

  • santé
  • religion
  • biométrie
  • opinions politiques

Principes RGPD

Licéité

Base légale obligatoire.

Transparence

Informer l'utilisateur.

Minimisation

Collecter uniquement le nécessaire.

Exactitude

Maintenir à jour.

Limitation de conservation

Ne pas conserver indéfiniment.

Sécurité

Protéger les données.

Droits des personnes

Droit d'accès

Voir ses données.

Droit de rectification

Corriger ses données.

Droit à l'effacement

Suppression.

Droit à la portabilité

Récupérer ses données.

Droit d'opposition

Refuser certains traitements.

Violation de données

Exemples :

  • fuite base RH
  • perte ordinateur portable
  • accès non autorisé

Procédure :

  1. Détecter
  2. Évaluer
  3. Notifier CNIL < 72h
  4. Informer les victimes
  5. Documenter
