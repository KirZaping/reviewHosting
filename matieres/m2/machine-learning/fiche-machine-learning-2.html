Partie 1 — Decision Tree Classification

2. Qu’est-ce qu’un arbre ?

Un arbre est une structure composée de nœuds.

Les trois types de nœuds sont :

Root node = nœud racine, point de départ de l’arbre
Internal nodes = nœuds internes, qui divisent les données
Terminal nodes / leaves = feuilles, qui donnent la prédiction finale

Dans un arbre de décision, chaque nœud pose une question du type :

petal length <= 2.45 ?
surface_m2 <= 85 ?
age_years <= 20 ?

Selon la réponse, l’observation part à gauche ou à droite jusqu’à arriver à une feuille.

3. Arbre de décision

Un Decision Tree est un algorithme de Machine Learning supervisé utilisable pour :

classification → prédire une classe
régression → prédire une valeur numérique

Exemples :

Classification : prédire setosa / versicolor / virginica
Régression : prédire le prix d’une maison

Son avantage principal est qu’il est facile à comprendre et à interpréter, car on peut suivre les décisions étape par étape.

4. Comment un arbre choisit un split ?

Un split est une séparation des données selon une variable et un seuil.

Exemple :

Day < 2.5
petal length <= 2.45
surface_m2 <= 120

L’algorithme teste plusieurs variables et plusieurs seuils, puis choisit le meilleur split.

Le principe est :

1. Tester plusieurs seuils possibles
2. Calculer un score pour chaque split
3. Choisir le split qui sépare le mieux les données
4. Répéter récursivement sur chaque sous-groupe

En classification, on utilise souvent des critères comme :

Gini impurity
Entropy

En régression, on utilise plutôt :

Variance reduction
Mean squared error reduction

5. Exemple simple de split en régression

Si on a :

Day Temp
1 10
2 15
3 20

L’arbre peut tester :

Day < 1.5
Day < 2.5

Pour chaque split, il calcule la variance dans les groupes créés. Le meilleur split est celui qui donne les groupes les plus homogènes.

À retenir :

Un bon split crée des groupes plus purs / plus homogènes.

6. Train / test split

Avant d’entraîner un modèle, il faut séparer les données en deux parties.

Train set = données utilisées pour apprendre
Test set = données utilisées pour évaluer

Exemple du Lab 3 :

from sklearn.model_selection import train_test_split
from sklearn.datasets import load_iris

iris = load_iris()
X = iris.data
y = iris.target

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.3,
    random_state=42
)

Ici :

70 % des données servent à l’entraînement
30 % des données servent au test

random_state=42 permet d’obtenir toujours le même découpage, donc des résultats reproductibles.

10. Lire un arbre de décision

Dans l’arbre affiché, chaque nœud contient généralement :

gini
samples
value
class

Gini

gini mesure l’impureté d’un nœud.

gini faible = nœud plutôt pur
gini élevé = nœud mélangé

Un nœud est pur si les observations qu’il contient appartiennent presque toutes à la même classe.

Samples

samples indique le nombre d’observations qui arrivent dans ce nœud.

Value

value indique la répartition des observations par classe.

Exemple :

value = [35, 0, 0]

Cela signifie :

35 setosa
0 versicolor
0 virginica

Class

class indique la classe majoritaire prédite par le nœud.

11. Arbre limité vs arbre complètement développé

Dans le Lab, on compare un arbre limité :

DecisionTreeClassifier(max_depth=2, random_state=42)

avec un arbre sans limite explicite :

DecisionTreeClassifier(random_state=42)

Un arbre limité est plus simple, plus lisible et risque moins de surapprendre.

Un arbre complètement développé peut devenir très précis sur le train set, mais il peut faire de l’overfitting.

12. Overfitting avec les arbres

L’overfitting arrive quand le modèle apprend trop précisément les données d’entraînement, y compris le bruit ou les cas particuliers.

Dans un arbre de décision, cela arrive souvent si l’arbre est trop profond.

Arbre trop profond → modèle complexe → risque d’overfitting
Arbre trop simple → modèle trop limité → risque d’underfitting

Pour limiter l’overfitting, on peut utiliser :

max_depth
min_samples_leaf
min_samples_split

21. Feature scaling

Les variables n’ont pas toutes la même échelle.

Exemple :

surface_m2 peut aller de 40 à 250
garage peut aller de 0 à 2
age_years peut aller de 0 à 100

Certains modèles sont sensibles à l’échelle, notamment KNN, car il utilise des distances.

On standardise donc les variables :

from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()

X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

Point important :

fit_transform sur le train set
transform seulement sur le test set

Il ne faut pas faire apprendre le scaler sur le test set, sinon il y a un risque de data leakage.

22. StandardScaler

StandardScaler transforme les variables pour qu’elles aient :

moyenne = 0
écart-type = 1

Formule :

z = (x - moyenne) / écart-type

Cette transformation rend les variables comparables entre elles.

Elle est particulièrement importante pour les modèles basés sur les distances :

KNN
SVM
K-Means
réseaux de neurones

Les arbres de décision, eux, sont beaucoup moins sensibles au scaling.

23. Fonction d’évaluation

Le Lab crée une fonction evaluate() pour entraîner un modèle, faire des prédictions et afficher ses performances.

Elle mesure :

MAE
R²
training time
prediction time

Code simplifié :

import time
from sklearn.metrics import mean_absolute_error, r2_score

def evaluate(model, X_tr, y_tr, X_te, y_te, name):
    start = time.time()
    model.fit(X_tr, y_tr)
    train_time = time.time() - start

    start = time.time()
    y_pred = model.predict(X_te)
    pred_time = time.time() - start

    mae = mean_absolute_error(y_te, y_pred)
    r2 = r2_score(y_te, y_pred)

    print(name)
    print("MAE:", mae)
    print("R2:", r2)
    print("Training time:", train_time)
    print("Prediction time:", pred_time)

L’intérêt de cette fonction est de comparer plusieurs modèles avec le même protocole.

24. MAE — Mean Absolute Error

La MAE mesure l’erreur moyenne absolue entre les vraies valeurs et les prédictions.

Formule intuitive :

MAE = moyenne des |prix réel - prix prédit|

Exemple :

MAE = 20 000 €

Cela signifie que le modèle se trompe en moyenne de 20 000 €.

À retenir :

MAE faible = meilleur modèle
MAE = 0 = prédiction parfaite

La MAE est facile à interpréter, car elle s’exprime dans la même unité que la variable cible.

25. R² — coefficient de détermination

Le R² mesure la part de variance expliquée par le modèle.

Interprétation :

R² proche de 1 → bon modèle
R² proche de 0 → modèle peu utile
R² négatif → modèle pire qu’une prédiction naïve

Exemple :

R² = 0.85

Cela signifie que le modèle explique environ 85 % de la variance de la variable cible.

Modèle 1 — KNN Regressor

27. Principe du KNN

Le KNN, ou K-Nearest Neighbors, prédit une valeur en regardant les k voisins les plus proches dans le train set.

En régression, il prédit souvent la moyenne des valeurs des voisins.

Exemple :

k = 5

Prix des 5 maisons les plus proches :
200 000, 210 000, 220 000, 230 000, 240 000

Prédiction ≈ 220 000 €

28. KNN est sensible au scaling

KNN utilise des distances.

Si une variable a des valeurs beaucoup plus grandes qu’une autre, elle domine le calcul de distance.

Exemple :

surface_m2 : 50 à 250
garage : 0 à 2

Sans scaling, surface_m2 pèsera beaucoup plus que garage.

C’est pourquoi le Lab utilise KNN avec les données standardisées :

from sklearn.neighbors import KNeighborsRegressor

knn = KNeighborsRegressor(n_neighbors=5)

res_knn = evaluate(
    knn,
    X_train_scaled,
    y_train,
    X_test_scaled,
    y_test,
    "KNN (k=5)"
)

29. Hyperparamètre k

Dans KNN, k correspond au nombre de voisins utilisés.

KNeighborsRegressor(n_neighbors=5)

Effet de k :

k petit → modèle très local, plus sensible au bruit
k grand → modèle plus stable, mais parfois trop général

Le Lab demande de tester plusieurs valeurs :

k = 1, 3, 5, 10, 20

Exemple de boucle :

for k in [1, 3, 5, 10, 20]:
    knn = KNeighborsRegressor(n_neighbors=k)
    evaluate(
        knn,
        X_train_scaled,
        y_train,
        X_test_scaled,
        y_test,
        f"KNN (k={k})"
    )

On choisit la valeur de k qui donne la meilleure performance, souvent la MAE la plus faible.

Modèle 2 — Decision Tree Regressor

30. Principe du Decision Tree Regressor

Un DecisionTreeRegressor est un arbre de décision utilisé pour prédire une valeur numérique.

Il fonctionne en divisant les données en régions.

Dans chaque région, il prédit une valeur constante, souvent la moyenne des valeurs du train set dans cette région.

Exemple :

Si surface_m2 <= 80 → prédiction moyenne = 220 000 €
Si surface_m2 > 80 → autre split

31. Entraîner un DecisionTreeRegressor

Dans le Lab, il faut créer un arbre avec :

max_depth = 5
random_state = 42

Code attendu :

from sklearn.tree import DecisionTreeRegressor

tree = DecisionTreeRegressor(max_depth=5, random_state=42)

res_tree = evaluate(
    tree,
    X_train,
    y_train,
    X_test,
    y_test,
    "Decision Tree (depth=5)"
)

Point important :

On utilise X_train et X_test non standardisés.

Pourquoi ? Parce que les arbres ne sont pas basés sur des distances. Ils cherchent des seuils de séparation, donc le scaling n’est pas nécessaire.

35. Améliorer un modèle

Le Lab propose d’aller plus loin en modifiant les hyperparamètres.

Pour KNN :

Pour Decision Tree :

Exemple :
