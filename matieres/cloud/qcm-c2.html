QCM 2 – Cloud Computing & Architectures SI

  1. Le modèle cloud favorise :

    • A. Des coûts fixes élevés
    • B. Le paiement à l’usage
    • C. Une plateforme non évolutive
    • D. Aucune élasticité

  2. L’expression « à la demande » en cloud signifie :

    • A. Provisionnement lent
    • B. Provisionnement rapide de ressources
    • C. Spécification uniquement locale
    • D. Une licence perpétuelle

  3. L’un des pièges classiques est de :

    • A. Confondre cloud et hébergement distant
    • B. Penser que le cloud nécessite aucune sécurité
    • C. Négliger les SLA
    • D. Externaliser uniquement le stockage

  4. La conteneurisation est plus légère que la virtualisation du fait :

    • A. D’un hyperviseur avancé
    • B. De l’absence d’OS invité
    • C. De la dépendance au matériel
    • D. D’une isolation réseau complète

  5. La virtualisation permet :

    • A. D’isoler les environnements
    • B. D’exécuter un seul OS
    • C. De réduire l’élasticité
    • D. D’éliminer les hyperviseurs

  6. Dans IaaS, quel élément est typiquement sous contrôle du fournisseur ?

    • A. Les applications du client
    • B. Le matériel et la virtualisation
    • C. Le middleware
    • D. La configuration serveur client

  7. Dans PaaS, le client conserve :

    • A. L’infrastructure matérielle
    • B. Le contrôle du runtime managé
    • C. La gestion du service réseau
    • D. La gestion des OS physiques

  8. L’un des avantages du PaaS est :

    • A. Une productivité accrue des développeurs
    • B. Une absence totale de lock-in
    • C. Une personnalisation hardware
    • D. Une disponibilité réseau garantie

  9. SaaS convient particulièrement à :

    • A. Applications personnalisées lourdes
    • B. Fonctions standards comme CRM
    • C. Développements internes complexes
    • D. Déploiement local exclusif

  10. L’un des risques de SaaS est :

    • A. Une personnalisation limitée
    • B. Une flexibilité totale
    • C. Une isolation matérielle accrue
    • D. Une gestion locale du middleware

  11. Le serverless facture généralement :

    • A. À la minute
    • B. À l’usage réel des ressources
    • C. À l’achat d’instances
    • D. À la licence annuelle

  12. La latence à froid en serverless correspond à :

    • A. Le temps de reprise d’une fonction inactive
    • B. Une dégradation réseau permanente
    • C. Un composant matériel
    • D. Une contrainte de sécurité

  13. Un inconvénient des architectures monolithiques est :

    • A. Une évolutivité indépendante
    • B. Un couplage fort
    • C. Une communication API native
    • D. Une disponibilité distribuée

  14. Les microservices permettent :

    • A. Une isolation complète des services
    • B. Une communication via API
    • C. Une structure monolithique
    • D. Une absence de réseau interne

  15. EDA est souvent utilisé pour :

    • A. Applications synchrones seulement
    • B. Découpler composants via événements
    • C. Héberger des VM uniquement
    • D. Un contrôle centralisé statique

  16. La scalabilité verticale se distingue par :

    • A. Ajouter des ressources à une instance existante
    • B. Ajouter plus d’instances horizontales
    • C. Une réduction de la charge
    • D. Une automatisation cloud native

  17. RPO dans les plans PCA/PRA correspond à :

    • A. Recovery Point Objective
    • B. Rapid Performance Outcome
    • C. Rate of Packet Output
    • D. Real Process Operation

  18. Dans le contexte cloud, la résilience implique :

    • A. Résistance aux pannes
    • B. Uniformité des coûts
    • C. Dépendance totale au fournisseur
    • D. Blocage du scaling

  19. Multi-Cloud désigne :

    • A. Une seule plateforme cloud
    • B. L’utilisation de plusieurs fournisseurs cloud
    • C. Un cloud interne uniquement
    • D. Un cloud local

  20. Infrastructure as Code (IaC) vise à :

    • A. Automatiser la gestion de l’infrastructure
    • B. Supprimer les descriptions versionnées
    • C. Augmenter le lock-in
    • D. Externaliser uniquement le stockage
